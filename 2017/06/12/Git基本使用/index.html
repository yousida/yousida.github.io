基本操作

设置username和email

创建版本库

在合适的地方创建文件夹作为版本库。

将文件夹加入Git管理git init，在文件夹中自动创建.git目录。

添加文件

  • 在版本库目录下新建的文件，使用git add readme.txt将文件添加到版本库。
  • 使用git commit -m "yourcomment"提交。

Tips：git add .添加全部文件。

查看版本库状态

git status

  • 修改的文件没有准备好被提交，即没有使用git add命令。
  • 修改的文件准备好被提交，即使用git add命令后未使用git commit命令。
  • 没有改动的文件。

使用git diff查看修改内容。

版本回退

使用git log查看提交日志。

Tips：commit代表提交版本号。

  • HEAD：当前版本。
  • HEAD^：上一版本。
  • HEAD~num：指定版本。

回退版本

  • git reset --hard HEAD^
  • git reset --hard commitId

使用git reflog查看命令日志。

39d00e3 HEAD@{6}: commit: comment

若回退最新版本，查看命令日志后git reset。

工作区和暂存区

工作区：电脑中看见的目录。

版本库：工作区中.git目录，其中有暂存区、分支和HEAD指针。

Tips：提交修改时，git add将文件提交至暂存区，git commit将暂存区内容提交至当前分支。

管理修改

Git管理修改，不是管理文件。

如对某一文件执行：

  • 修改
  • git add
  • 修改
  • git commit -m

第二次修改没有提交，即git commit -m提交暂存区改动。

撤销修改

git checkout -- Test.txt撤销工作区修改。

  • Test.txt修改后未放到暂存区，撤销修改后和版本库一致。
  • Test.txt提交至暂存区后修改，撤销修改后和暂存区一致。

git reset HEAD Test.txt撤销暂存区修改。

删除文件

本地目录中物理删除文件后，使用git rm Test.txt删除仓库中的文件后git commit -m "yourcomment"提交修改。

本地目录中删除文件后，想要恢复，使用git checkout -- Test.txt撤销修改(版本库中存在被删除文件记录)。

远程仓库

本地仓库和远程仓库通过SSH加密。

  • 创建SSH Key，在用户主目录下，查看是否存在.SSH目录以及目录中是否存在id_rsa（私钥）和id_rsa.pub（公钥），如果没有，使用ssh-keygen -t rsa -C "youreamil"创建，一路回车即可。
  • 在远程仓库中添加SSH Key。

本地仓库推送至远程仓库

  • 创建新的远程仓库（如GitHub上创建仓库）。
  • 在本地中，使用
    git remote add origin git@github.com:username/RepoName.git进行关联。
  • 使用git push --set-upstream origin master将本地仓库中master的内容推送至远程分支。

Tips：–set-upstream将本地master分支推送至远程master分支（若不存在，则新建），并关联起来，以后推送只需git push origin命令。

克隆远程仓库

进入对应目录后使用git clone git@github.com:username/RepoName.git命令。

分支

图示

Git将每个提交版本串成时间线，这条时间线即为一个分支。

Head指向当前分支master，master指向最新提交版本。

当新建分支dev，Git新建指针dev，指向与master相同的提交版本，Head指向指针dev。

对新建分支进行修改，master指针不变，移动dev指针。

在dev分支上完成任务，合并至master分支，将master指向dev最新版本，并将Head指针指向master指针。

删除分支，即删除分支指针。

命令

创建分支git branch dev

切换分支git checkout dev

查看分支git branch

合并分支git merge dev

删除已合并分支git branch -d dev

删除未合并分支git branch -D dev
