RPC

RPC即远程调用，两台服务器A和B分别部署不同应用a和b，当A服务器想要调用B服务器上b应用提供的函数或方法时，不能直接调用，需要通过网络进行远程调用。

其中，使用什么协议（Http、Tcp、Udp…），传输什么格式（Xml、Json…）可以自定义。

Dubbo时Rpc远程调用方案。

角色

Provider：暴露服务的服务提供者。

Consumer：调用远程服务的服务消费者。

Registry：服务注册与发现的注册中心。

Monitor：统计服务调用次数和调用时间的监控中心。

流程

  • 服务容器负责启动、加载和运行服务提供者。
  • 服务提供者在启动时，向注册中心注册自己提供的服务。
  • 服务消费者在启动时，向注册中心订阅自己所需服务。
  • 注册中心返回服务提供者地址列表给服务消费者。
  • 服务消费者，从提供者地址列表中，选择进行调用。
  • 服务提供者和服务消费者，在内存中累计调用次数和调用时间，定时发送到监控中心。

注册中心

通过将服务统一管理，可以有效优化服务发布／使用的流程和管理，存在以下注册中心：

  • Multicast注册中心
  • Zookeeper注册中心
  • Redis注册中心
  • Simple注册中心
