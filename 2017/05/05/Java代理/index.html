基础

代理类在运行时生成。

java.lang.reflect.Proxy

动态代理机制的主类，提供一种静态方法为接口动态的生成代理类。

java.lang.reflect.InvocationHandler

调用处理器接口，自定义invoke方法，此方法中实现对目标类的代理访问。

生成代理类需要一个实现调用处理器接口的对象。

步骤

  • 通过实现InvocationHandler接口创建调用处理器。
  • 通过为Proxy类指定ClassLoader对象和一组Class创建动态代理类对象。
  • 通过反射机制获得动态代理类的构造函数，其唯一参数调用处理器。
  • 通过构造函数创建动态代理类实例，构造时调用处理器作为参数传入。

特点

  • 每个实例关联一个调用处理器对象，可以通过Proxy的静态方法getInvocationHandler获得调用处理器对象。
  • 在代理类实例调用接口声明的方法时，最终由调用处理器的invoke方法执行。
  • 当代理的一组接口有相同的方法声明时，代理类从排在前面的接口中获取方法对象分配给调用处理器。
